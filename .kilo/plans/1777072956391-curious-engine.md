# Fix Plan: Review Findings Cleanup

## Goal
Address the five review findings from the branch diff with minimal, behavior-preserving changes:
- restore project-load compatibility for omitted `pctMin`
- constrain NPC image lookup to the intended `images/` folder
- make the image window reusable after close
- add a failure boundary for drag-and-drop processing
- remove committed personal IDE metadata and prevent it from reappearing

## Scope
Limit implementation to the issues called out in the final review. Avoid unrelated refactors or broader repository cleanup unless a fix directly requires it.

## Assumptions
- Keep current UX and status-message patterns where possible.
- Prefer the smallest correct fix over introducing new abstractions.
- Preserve existing valid image lookup behavior: `Name (EditorId)` first, then `Name`.
- Preserve the current shared `.idea` files except for the user/plugin-specific Material Theme metadata.

## Work Items

### 1. Restore legacy `pctMin` default behavior
**Files:**
- `src/BS2BG.Core/Serialization/ProjectFileService.cs`
- `tests/BS2BG.Tests/ProjectFileServiceTests.cs`

**Plan:**
- Change `SetSlider.PercentMin` loading so an omitted `pctMin` defaults to `100`, matching the model default and the existing missing-default omission rules.
- Leave `PercentMax` fallback at `100`.
- Add a regression test that loads JSON with omitted `pctMin` and asserts:
  - `PercentMin == 100`
  - `PercentMax == 100`
  - saved output remains compatible with the intended default semantics
- Re-run the existing round-trip fixture coverage to confirm no unintended serialization changes.

### 2. Constrain NPC image lookup to `images/`
**Files:**
- `src/BS2BG.App/Services/NpcImageLookupService.cs`
- `tests/BS2BG.Tests/NpcImageLookupServiceTests.cs`

**Plan:**
- Add a helper that builds candidate file names safely from NPC data.
- Reject path-like or rooted inputs coming from `npc.Name` / `npc.EditorId` before probing the file system.
- Normalize the candidate path with `Path.GetFullPath` and assert it stays under the normalized `images` directory before calling `File.Exists`.
- Keep the current extension order and fallback behavior for valid names.
- Add regression tests for:
  - directory traversal input returning `null`
  - rooted/path-like input returning `null`
  - normal NPC-specific lookup still winning over the name-only fallback

**Implementation note:**
Use a narrow containment check rather than broad name rewriting so legitimate NPC names continue to work.

### 3. Recreate the image window after close
**Files:**
- `src/BS2BG.App/Services/WindowImageViewService.cs`
- `tests/BS2BG.Tests/WindowImageViewServiceTests.cs`

**Plan:**
- When creating the cached image window, subscribe to `Closed` and clear cached references (`window`, `imageControl`).
- Preserve current owner attachment, sizing, `Topmost`, and `CreateBitmap` behavior.
- Add a headless regression test that:
  - shows an image window
  - closes it
  - shows another image again
  - verifies the second show path recreates a usable window and does not fail

**Implementation note:**
Prefer clearing the closed instance over changing close behavior to `Hide()` unless tests show Avalonia-specific constraints.

### 4. Add a drag-and-drop failure boundary
**Files:**
- `src/BS2BG.App/ViewModels/MainWindowViewModel.cs`
- `src/BS2BG.App/Views/MainWindow.axaml.cs` if needed
- `tests/BS2BG.Tests/MainWindowViewModelTests.cs` and/or a focused headless UI test

**Plan:**
- Add a top-level failure boundary around dropped-file processing so exceptions do not escape the `async void` UI event path.
- Preferred minimal approach: catch in `HandleDroppedFilesAsync` so status formatting remains in the view model and no broad API change is needed.
- If that still leaves an uncovered gap around path extraction in `OnDrop`, add a narrow catch there and delegate to a small view-model helper.
- Report failures with status text consistent with existing patterns, e.g. `"Dropped file processing failed: ..."`.
- Add the cheapest meaningful regression test available after implementation:
  - preferred: a `MainWindowViewModelTests` case if the exception path can be triggered cleanly
  - fallback: a focused headless test that exercises the UI-side boundary

**Implementation note:**
Many current open/import collaborators already catch their own file errors, so this is partly a defensive UI-safety fix. Keep it surgical.

### 5. Remove committed personal IDE metadata
**Files:**
- `.idea/.idea.BS2BG/.idea/material_theme_project_new.xml`

**Plan:**

**Recommended ignore shape:**
