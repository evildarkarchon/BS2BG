# Plan: Fix Window Image Bitmap Disposal Leak

## Goal
Fix the resource leak in `src/BS2BG.App/Services/WindowImageViewService.cs` where replacing or closing an NPC image view leaves the previous Avalonia `Bitmap` undisposed.

## Scope
- Modify only `WindowImageViewService` production code.
- Add or update tests only if needed to validate disposal behavior without making the test brittle.
- Do not change unrelated image lookup, NPC import, or window ownership behavior.

## Implementation Steps
1. In `ShowImage`, capture and dispose any existing `imageControl.Source` before assigning the newly created bitmap.
2. In the image window `Closed` handler, detach and dispose the current image source before setting `imageControl` and `window` to `null`.
3. Add a small private helper such as `DisposeImageSource(Image? image)` that:
   - Returns immediately when the image is `null`.
   - Stores `image.Source` in a local variable.
   - Sets `image.Source = null` before disposal.
   - Disposes the old source when it implements `IDisposable`.
4. Keep `CreateBitmap` behavior unchanged so unsupported or unreadable images still return `null` safely.

## Verification
1. Run `dotnet test BS2BG.sln`.
2. Confirm the existing `WindowImageViewServiceTests.ShowImageRecreatesWindowAfterClose` still passes.
3. If a lightweight assertion is practical, extend image-view tests to verify closing/replacing clears or disposes sources via observable state; otherwise rely on the production helper and full test suite.

## Expected Outcome
Repeated NPC image previews and image window closes no longer retain unmanaged bitmap resources, while existing image display and invalid-image handling behavior remains unchanged.
