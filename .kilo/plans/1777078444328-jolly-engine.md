# Gate shell commands on aggregate busy state

## Context

A prior change (`glowing-lagoon`) added `MainWindowViewModel.IsAnyBusy = IsBusy || Templates.IsBusy || Morphs.IsBusy` and bound the header progress bar to it. However, the `CanExecute` predicates on the shell-level `AsyncRelayCommand`s still only inspect the shell's own `IsBusy`:

- `MainWindowViewModel.cs:587` — `CanRunShellCommand() => !IsBusy`
- `MainWindowViewModel.cs:589` — `CanSaveProject() => !IsBusy && ...`
- `MainWindowViewModel.cs:591` — `CanExportBosJson() => !IsBusy && ...`
- `MainWindowViewModel.cs:593-599` — `CanExportBodyGenInis()` predicate (same pattern)
- `MainWindowViewModel.cs:135` — `ShowAboutCommand` uses inline `() => !IsBusy`

Long-running child operations (XML preset imports on `TemplatesViewModel`, NPC imports on `MorphsViewModel`) only set their own `IsBusy`. The shell's `IsBusy` stays `false`, so New / Open / Save / Save As / Export BoS JSON / Export BodyGen INIs / About all remain enabled while a child import is in flight.

Concrete failure modes this enables:

1. User drags in NPC `.txt` files → `Morphs.ImportNpcFilesAsync` starts, `Morphs.IsBusy = true`. User clicks **New Project** before the parse finishes. `NewProjectAsync` replaces the `ProjectModel`. When the import completes, it mutates the project into the freshly-cleared state.
2. Same race with **Open Project**: a loaded project can be clobbered by a still-running import.
3. **Save** / **Save As** during an import captures a half-populated `ProjectModel` snapshot.
4. **Export** predicates read `project.SliderPresets.Count` at a moment that a concurrent import is mutating the collection.

The `PropertyChanged` wiring for `IsAnyBusy` already exists (`OnChildBusyChanged` at `MainWindowViewModel.cs:561-568`) but only raises `IsAnyBusy`; it does not re-raise the shell commands' `CanExecuteChanged`, so even after switching the predicates, WPF/Avalonia bindings would not re-query until something else triggered `RaiseCommandStatesChanged`.

## Goal

Gate every shell-level command on `IsAnyBusy` instead of `IsBusy`, and ensure the shell commands' `CanExecuteChanged` is re-raised whenever a child VM's `IsBusy` toggles, so menu items / toolbar buttons disable reliably while any import is in flight.

## Approach

Pure VM change. Swap `!IsBusy` for `!IsAnyBusy` in the four private predicates and the inline `ShowAboutCommand` predicate, then extend `OnChildBusyChanged` to also call `RaiseCommandStatesChanged()` so the command bindings re-query.

### Changes — `src/BS2BG.App/ViewModels/MainWindowViewModel.cs`

1. **Line 135** — replace:
   ```csharp
   ShowAboutCommand = new RelayCommand(ShowAbout, () => !IsBusy);
   ```
   with:
   ```csharp
   ShowAboutCommand = new RelayCommand(ShowAbout, () => !IsAnyBusy);
   ```

2. **Line 587** — replace:
   ```csharp
   private bool CanRunShellCommand() => !IsBusy;
   ```
   with:
   ```csharp
   private bool CanRunShellCommand() => !IsAnyBusy;
   ```

3. **Line 589** — replace:
   ```csharp
   private bool CanSaveProject() => !IsBusy && (project.IsDirty || CurrentProjectPath is null);
   ```
   with:
   ```csharp
   private bool CanSaveProject() => !IsAnyBusy && (project.IsDirty || CurrentProjectPath is null);
   ```

4. **Line 591** — replace:
   ```csharp
   private bool CanExportBosJson() => !IsBusy && project.SliderPresets.Count > 0;
   ```
   with:
   ```csharp
   private bool CanExportBosJson() => !IsAnyBusy && project.SliderPresets.Count > 0;
   ```

5. **Lines 593-599** — replace the first `!IsBusy` in `CanExportBodyGenInis` with `!IsAnyBusy`.

6. **`OnChildBusyChanged` (lines 561-568)** — after raising `IsAnyBusy`, also invoke `RaiseCommandStatesChanged()` so every shell command re-queries its predicate when a child flips `IsBusy`:
   ```csharp
   private void OnChildBusyChanged(object? sender, PropertyChangedEventArgs args)
   {
       if (args.PropertyName == nameof(TemplatesViewModel.IsBusy)
           || args.PropertyName == nameof(MorphsViewModel.IsBusy))
       {
           this.RaisePropertyChanged(nameof(IsAnyBusy));
           RaiseCommandStatesChanged();
       }
   }
   ```

No other files require changes:
- `IsBusy` setter (lines 190-201) already calls `RaiseCommandStatesChanged()` and raises `IsAnyBusy`; behavior when the shell itself flips busy is unchanged.
- XAML bindings keep referencing `IsAnyBusy` / command `CanExecute`; no view edits.

### Not changed

- `Undo` / `Redo` predicates are intentionally not touched — they already only gate on `undoRedo.CanUndo` / `CanRedo`. Undo during an import is not one of the concurrency hazards called out; applying busy-gating there is a separate policy call.
- `FocusGlobalSearchCommand`, `OpenCommandPaletteCommand`, `CloseCommandPaletteCommand`, `RunCommandPaletteItemCommand` — UI navigation, not state-mutating.
- The command palette entries in `RegisterCommandPaletteItems` forward to the same `ICommand` instances, so they inherit the tightened gating automatically.

## Tests — `tests/BS2BG.Tests/MainWindowViewModelTests.cs`

Existing tests (`IsAnyBusyRaisesWhenTemplatesIsBusyToggles`, `…WhenMorphsIsBusyToggles`) already drive `Templates.ImportPresetsAsync` / `Morphs.ImportNpcsAsync` through `MainWindowViewModel` and capture `PropertyChanged` events. We piggyback on that pattern to assert command gating mid-flight.

Add three focused tests (a fourth is optional):

1. **`NewProjectCommandIsDisabledWhileTemplatesAreBusy`**
   - Create VM, subscribe to `PropertyChanged`.
   - Inside the handler, when `args.PropertyName == nameof(IsAnyBusy)` *and* `viewModel.IsAnyBusy` is `true`, capture `viewModel.NewProjectCommand.CanExecute(null)` into a list.
   - `await viewModel.Templates.ImportPresetsAsync(...)` — the empty picker path still flips `Templates.IsBusy` on and off.
   - Assert the captured list contains at least one `false` entry, and that `NewProjectCommand.CanExecute(null)` is `true` again after the await.

2. **`SaveProjectCommandIsDisabledWhileMorphsAreBusy`**
   - Same shape, driving `Morphs.ImportNpcsAsync`. Pre-seed the project with a preset so `CanSaveProject` would otherwise return `true`. Capture `SaveProjectCommand.CanExecute(null)` inside the `IsAnyBusy→true` event and assert at least one `false` sample.

3. **`ExportBosJsonCommandIsDisabledWhileTemplatesAreBusy`**
   - Pre-seed one `SliderPreset` so `CanExportBosJson` would otherwise be `true`. Drive `Templates.ImportPresetsAsync`. Capture `ExportBosJsonCommand.CanExecute(null)` on `IsAnyBusy→true`; assert `false` was observed.

4. *(Optional)* **`ShowAboutCommandIsDisabledWhileChildIsBusy`** — same pattern covering the inline predicate change at line 135.

Test helper shape (shared inline pattern — do not factor into a helper yet, each test is ~15 lines):

```csharp
var disabledDuringBusy = new List<bool>();
viewModel.PropertyChanged += (_, args) =>
{
    if (args.PropertyName == nameof(MainWindowViewModel.IsAnyBusy) && viewModel.IsAnyBusy)
        disabledDuringBusy.Add(viewModel.NewProjectCommand.CanExecute(null));
};
await viewModel.Templates.ImportPresetsAsync(TestContext.Current.CancellationToken);
disabledDuringBusy.Should().NotBeEmpty();
disabledDuringBusy.Should().OnlyContain(canExecute => !canExecute);
viewModel.NewProjectCommand.CanExecute(null).Should().BeTrue();
```

Note: depending on the order `OnChildBusyChanged` raises `IsAnyBusy` vs. `RaiseCommandStatesChanged()`, the `PropertyChanged` listener observes the command after `RaiseCommandStatesChanged` has already fired `CanExecuteChanged`, but the listener only cares that `CanExecute(null)` returns the post-toggle value — which it will, because the predicate reads `IsAnyBusy` directly and `Templates.IsBusy` is already `true` at this point.

- `dotnet test tests/BS2BG.Tests/BS2BG.Tests.csproj` — all existing tests plus the three (optional four) new tests pass.

  2. While the import runs, observe that **File → New**, **Open**, **Save**, **Save As**, **Export BoS JSON**, **Export BodyGen INIs** are all greyed out.
  3. After import completes, menu items re-enable.

## Risk / Rollback

- **Risk: low.** Predicate change is monotonic (commands become *less* available, never more). No new allocations, no thread-affinity implications — `IsAnyBusy` is a pure getter.
- **Risk: UX regression** — the About dialog being disabled during an import is a minor tightening. If the team prefers to keep About always available, drop change (1) from the list; the correctness-critical fixes are (2)-(6).
- **Rollback:** revert the single file.

## Out of Scope

- Undo/Redo gating during child imports (current behavior preserved deliberately).
- Making `Templates` / `Morphs` own commands consult each other's busy state — each child only gates on its own `IsBusy`, which is already correct because children mutate independent slices of the project.
- Replacing the cooperative busy-flag scheme with a single re-entrant gate / CancellationToken pipeline — larger refactor, separate proposal.
- Any view / XAML changes.
