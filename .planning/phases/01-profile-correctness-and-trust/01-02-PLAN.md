## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `.jbs2bg` project JSON -> Core model | User-controlled project files can contain arbitrary `Profile` strings. |
| Core catalog -> App ViewModel | App code uses Core catalog results to decide whether fallback information is shown. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | Tampering | `ProjectFileService.LoadFromString` | mitigate | Treat profile names as plain strings; add tests that unbundled names are preserved, not rewritten to a bundled profile silently. |
| T-01-05 | Spoofing | `TemplateProfileCatalog.ContainsProfile` | mitigate | Use exact bundled display-name matching with `StringComparison.OrdinalIgnoreCase`; unknown names remain distinguishable from bundled profiles. |
| T-01-06 | Denial of Service | Unknown project profile names | accept | Unknown names do not block load/generation; fallback remains non-throwing and visible in later UI work. |
</threat_model>
