## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline gaps, splitter-adjacent padding, compact inspector details |
| sm | 8px | Control gaps, label-to-field spacing, neutral info banner internal rhythm |
| md | 16px | Default group spacing, profile selector separation from actions |
| lg | 24px | Section separation where the fallback info panel sits between toolbar and preview |
| xl | 32px | Major horizontal region separation when adding any new profile metadata area |
| 2xl | 48px | Reserved for future page-level breaks; do not introduce in Phase 1 unless layout grows |
| 3xl | 64px | Reserved; no Phase 1 usage |

Exceptions: none for new Phase 1 UI. Preserve existing layout measurements unless the touched area is being edited; new fallback/profile UI must use 4px multiples only.

## Typography

Use exactly these four sizes and two weights for touched UI:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | Regular 400 | 1.5 |
| Label | 13px | Regular 400 | 1.4 |
| Heading | 18px | SemiBold 600 | 1.2 |
| Monospace Output | 12px | Regular 400 | 1.4 |

Rules:
- Keep existing output text boxes on monospace (`Consolas`) at 12–13px as already used in `MainWindow.axaml`.
- Use SemiBold only for section labels and the app title; do not bold profile names inside normal copy.
- Neutral fallback information must use body text, not warning/emphasis typography.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Light `#F7F9FC`; Dark `#11151A` | Window background and primary workspace surface via `BS2BGWindowBackgroundBrush` |
| Secondary (30%) | Fluent chrome/card resources (`SystemControlBackgroundChromeMediumBrush` / standard control surfaces) | Cards, popup surfaces, list/textbox/control backgrounds |
| Accent (10%) | Light `#005FB8`; Dark `#7CC4FF` | Focus states, selected control affordances, links only if introduced |
| Destructive | Fluent error resources (`SystemControlErrorTextForegroundBrush`) | Validation errors and destructive action text only |

Accent reserved for: keyboard focus, selected control affordances, and optional inline help/link affordance. Do not use accent for the unresolved-profile fallback panel background; it must read as neutral information, not urgency.

Neutral fallback information colors:
- Light: background `#EEF6FF`, border `#8CBFE8`, foreground `#17324D`.
- Dark: background `#102A43`, border `#4E8DBD`, foreground `#D8ECFF`.
- Add these as named theme resources if implemented; do not reuse `BS2BGWarning*` brushes.

## Phase 1 Interaction Contract

### Profile selector
- The profile selector remains in the Templates toolbar, immediately before `Omit Redundant Sliders` and `Generate Templates`.
- Changing the selector updates the selected preset's profile, refreshes missing-default rows, refreshes template preview, refreshes selected BoS JSON, and clears generated template text.
- Imported BodySlide XML presets use the profile currently selected by the user at import time. Do not infer from file path, game folder, or slider names.
- Legacy projects without `Profile` map silently through `isUUNP`: `true` => `Skyrim UUNP`; `false` => `Skyrim CBBE`.
- Projects containing an unbundled saved profile name preserve that exact name for save/load until the user explicitly selects a bundled profile.

### Neutral fallback information
- Show the neutral fallback text only when a selected preset has an unbundled saved profile name and preview/generation would otherwise use bundled fallback calculation rules silently.
- Place the neutral info panel directly below the profile toolbar and above the template preview/status area.
- The primary focal point remains the selected preset's `Profile` selector; the fallback panel is secondary and must not visually overpower `Generate Templates` or the preview.
- The panel must not block generation, export, copy, preview, or profile changes.
- The panel must not use warning words, warning iconography, warning colors, modal dialogs, or confirmation prompts.
- Hide the panel immediately after the user chooses a bundled profile for the selected preset.

### Explicit non-goals for Phase 1 UI
- Do not add general warnings for custom/unprofiled body mods.
- Do not add slider-name mismatch warnings.
- Do not label Fallout 4 CBBE as experimental anywhere in the main workflow.
- Do not add profile auto-detection from paths, game folders, XML names, or slider overlap.
- Do not introduce custom profile management UI; that belongs to Phase 4.

### Accessibility and testability
- Every new AXAML root/data template must keep compiled bindings valid with `x:DataType`.
- New fallback text must be exposed through a bound ViewModel property so ViewModel tests can assert visibility and exact copy.
- If the fallback panel is added to AXAML, set an automation name such as `Profile fallback information`.
- ViewModel/headless UI coverage must include profile selection updates, imports using the selected profile, unresolved saved profile preservation, visible neutral fallback information, and absence of mismatch warnings for custom/unprofiled slider names.
