# Phase 02 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — existing Avalonia 12 desktop app, not a web/shadcn frontend |
| Preset | not applicable |
| Component library | Avalonia 12 FluentTheme controls (`Window`, `TabControl`, `ListBox`, `Popup`, `ComboBox`, `TextBox`, `Button`, `GridSplitter`) |
| Icon library | none currently used; do not add iconography for this phase unless an existing command already has an icon pattern |
| Font | Avalonia FluentTheme system UI font; use `Consolas` only for generated text/JSON/INI previews |

Source: existing `App.axaml`, `ThemeResources.axaml`, and `MainWindow.axaml`. Preserve the current MainWindow visual language: dense utility layout, tabs for Templates/Morphs, split panes, text-first commands, theme-aware Fluent resources, compiled bindings, and ReactiveUI ViewModel command wiring.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | GridSplitter thickness, compact separators, dense row padding |
| sm | 8px | Button gaps, popup padding, filter value spacing, inline control margins |
| md | 16px | Default page padding, toolbar gaps, status-banner padding (`BS2BGPagePadding`) |
| lg | 24px | Section grouping inside popups or panes when a stronger break is needed |
| xl | 32px | Major pane section breaks only; avoid inside dense table/list rows |
| 2xl | 48px | Reserved for modal/dialog body separation; not expected in the main Morphs grid |
| 3xl | 64px | Reserved for empty-state vertical breathing room; not for normal controls |

Exceptions: preserve existing MainWindow micro-spacing where already present, but new Phase 2 UI must use the scale above. Per-column filter popups use `Padding=8`, `Width=240`, `MaxHeight=280`; scope selector spacing uses 8px from adjacent action buttons. Minimum pointer target for new icon-free buttons remains the default Avalonia `Button` height; do not create custom sub-32px click targets.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 | 1.5 |
| Label | 12px | 400 | 1.4 |
| Heading | 18px | 600 | 1.2 |
| Display | 13px | 400 | 1.4 |

Constraints:
- Use exactly two text weights for new Phase 2 UI: regular `400` and semibold `600`.
- Use `Heading` for pane titles such as `Individual NPCs` and `Imported NPCs`; do not add oversized marketing-style headings.
- Use `Display` with `Consolas` for generated text, JSON, INI, row-id diagnostics if surfaced, and other machine-readable previews.
- Filter popup helper text and value counts use `Label`; command labels and status text use `Body`.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Light `#F7F9FC`, Dark `#11151A` | Window background and main workspace surface via `BS2BGWindowBackgroundBrush` |
| Secondary (30%) | Fluent chrome resources (`SystemControlBackgroundChromeMediumBrush`, `SystemControlForegroundBaseLowBrush`) | Popups, pane borders, list separators, filter dropdown surfaces |
| Accent (10%) | Light `#005FB8`, Dark `#7CC4FF` | Keyboard focus, active filter indicators, selected scope emphasis, command-palette focus states |
| Destructive | Light `#C42B1C`, Dark `#FF99A4` | Destructive all-scope confirmation text/buttons only; validation/error text continues to use `SystemControlErrorTextForegroundBrush` |

Accent reserved for: active per-column filter markers, focused filter controls, selected bulk-scope indicator, command palette focus, and positive confirmation affordances. Do not color every interactive button accent; ordinary workflow commands remain default Fluent buttons.

Semantic information colors already exist and must be reused:
- Info: `BS2BGInfoBackgroundBrush`, `BS2BGInfoBorderBrush`, `BS2BGInfoForegroundBrush` for neutral workflow/fallback/persistence notices.
- Warning: `BS2BGWarningBackgroundBrush`, `BS2BGWarningBorderBrush`, `BS2BGWarningForegroundBrush` for non-destructive limits such as target preset count warnings.
- Error/destructive: Fluent error resources plus the destructive values above for confirmed all-row destructive actions only.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `Fill Visible Empty` |
| Empty state heading | `No NPCs match the current filters` |
| Empty state body | `Clear one or more filters to show hidden NPCs, or change the bulk scope before running an action.` |
| Error state | `This workflow preference could not be saved. BS2BG will continue using defaults for this session.` |
| Destructive confirmation | `Clear all NPC assignments`: `This will clear assignments for every NPC, including rows hidden by filters. You can undo this action. Continue?` |

Additional required labels and status copy:
- Bulk scope selector label: `Scope`.
- Scope options: `All`, `Visible`, `Selected`, `Visible Empty`.
- When filters are active, default displayed scope for routine bulk operations: `Visible`.
- Random fill primary scope: `Visible Empty`.
- Active filter badge: `{Column}: {count} selected` for checklist filters; use `Race: 3 selected`, `Preset: 1 selected`, etc.
- Filter popup search placeholder: `Search {column}` (`Search mods`, `Search names`, `Search editor IDs`, `Search form IDs`, `Search races`, `Search assignment states`, `Search presets`).
- Filter popup clear button: `Clear`.
- Non-blocking undo-prune status: `Undo history trimmed to keep large workflows responsive.`
- Hidden selection status when applicable: `{visible} visible, {selected} selected ({hiddenSelected} hidden by filters)`.
- Preference persistence is best-effort: do not show modal errors for preference load/save failure.

## Phase 2 Interaction Contract

### Preference Persistence

- Persist `Omit Redundant Sliders` as local app preference only; visual state hydrates on startup before the user generates templates.
- Remember five separate folder channels: project open/save, BodySlide XML import, NPC text import, BodyGen INI export, and BoS JSON export.
- Preference load/save failure never blocks startup, import, generation, save, or export. At most, show the non-blocking error-state copy in the existing shell status region.
- NPC filter/search state is session-only and must reset on restart.

### NPC Filtering

- Add per-column checklist filters for: mod/master, name, editor ID, form ID, race, assignment state, and preset-related values.
- Each column filter uses the existing race-filter popup pattern: column header/button, bottom popup, search textbox, multi-select checklist, `Clear` button.
- Checklist changes apply immediately. Popup search text filters only popup values and does not mutate NPC rows.
- Global/free-text NPC search is debounced before visible rows rebuild.
- Active filters must be visually discoverable from the column header/button using the accent color and active-filter badge text.
- Empty filtered views show the empty-state copy in the list area; they must not look like data loss.
- Hidden selected NPCs remain selected when filters change. Surface hidden selection count in status text when non-zero.

### Bulk Operation Scope

- Place one explicit `Scope` selector near Morphs bulk action buttons, before `Fill Visible Empty`, `Clear Assignments`, remove/clear, and selected-assignment actions.
- Scope options are exactly `All`, `Visible`, `Selected`, `Visible Empty`.
- When filters are active, routine bulk operations default to `Visible`.
- Random fill defaults to `Visible Empty` and its primary CTA label is `Fill Visible Empty`.
- Command status after a scoped action must include affected count and scope, e.g. `Assigned presets to 42 visible empty NPCs.`
- Destructive all-scope actions require confirmation; visible/selected actions rely on explicit scope labels and undo/redo, not repeated prompts.

### Undo/Redo Feedback

- Existing `Ctrl+Z`, `Ctrl+Y`, Edit menu, and shell status patterns remain the primary undo/redo UI.
- Each bulk operation is one undoable user action; status copy should describe the action as one operation, not per-row work.
- When bounded undo history prunes old entries, show the non-blocking undo-prune status in the shell status region.
- Undo/redo UI must never imply hidden filtered rows were untouched unless the selected scope actually excluded them.

### Accessibility and Keyboard

- Every new filter button, scope selector, popup search box, and destructive confirmation control must set `automation:AutomationProperties.Name`.
- Preserve existing keyboard shortcuts. Do not repurpose `Ctrl+F`, `Ctrl+Z`, `Ctrl+Y`, or command-palette gestures.
- Filter popups must be light-dismissable and Escape-dismissable where practical.
- Tab order: global/pane search → scope selector → primary bulk action → secondary actions → list/grid.
- New AXAML roots and DataTemplates must declare `x:DataType`; use compiled bindings by default.
