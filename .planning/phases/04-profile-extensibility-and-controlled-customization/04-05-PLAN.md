<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user action/file picker → App ViewModel | User-selected JSON and save destinations enter App orchestration. |
| App ViewModel → profile store | Only validated definitions may be written to local profile storage. |
| bundled catalog entry → editor | Read-only bundled data must be copied before edit. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05-01 | Tampering | ProfileManagerViewModel.ImportProfileCommand | mitigate | Validate imported JSON with Core service before store write or catalog refresh. |
| T-04-05-02 | Elevation of Privilege | ProfileManagerViewModel bundled rows | mitigate | Disable edit/delete/save for `ProfileSourceKind.Bundled`; only copy-as-custom is allowed. |
| T-04-05-03 | Tampering | ProfileEditorViewModel.SaveProfileCommand | mitigate | Gate save on validation blockers and route writes through UserProfileStore, not direct ViewModel file I/O. |
</threat_model>
