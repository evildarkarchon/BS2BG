<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| embedded project profiles → local user profile store | Shared projects may contain profiles that conflict with or attempt to replace local data. |
| project-scoped overlay → generation catalog | Embedded project profiles can affect lookup for the active project only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06-01 | Tampering | MainWindowViewModel project open | mitigate | Snapshot local profiles, collect all decisions before mutation, and abort/rollback on cancel or store-write failure. |
| T-04-06-02 | Spoofing | TemplateProfileCatalogService overlay | mitigate | Allow project-scoped local-custom override only for active project custom names; reject bundled-name collisions. |
| T-04-06-03 | Repudiation | Rename Project Copy during open | mitigate | Mark loaded project dirty after open when profile references are renamed; do not hide project changes as clean. |
</threat_model>
