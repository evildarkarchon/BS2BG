<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| embedded project profiles → local user profile store | Shared projects may contain profiles that conflict with local data. |
| recovery UI action → project model | Explicit actions can mutate preset profile references or local store. |
| diagnostics → mutation commands | Read-only findings must not auto-fix without user intent. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06-01 | Tampering | MainWindowViewModel project open | mitigate | Detect same-name different embedded/local profiles and prompt before replacing, renaming, or keeping data. |
| T-04-06-02 | Repudiation | Templates remap action | mitigate | Record undo/redo and dirty state for remap/adoption changes so user can reverse explicit recovery. |
| T-04-06-03 | Tampering | Diagnostics recovery actions | mitigate | Diagnostics remain read-only until explicit command invocation; import matching uses internal display-name exact match. |
</threat_model>
