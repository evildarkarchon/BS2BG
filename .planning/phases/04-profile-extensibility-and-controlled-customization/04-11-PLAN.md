<objective>
Close Phase 4 verification gaps in GUI project sharing saves and embedded/local profile conflict rename validation.

Purpose: Core serialization already supports `ProjectSaveContext`, but the app save path bypasses it, so normal GUI saves can reference local custom profiles without embedding definitions. Conflict rename validation also uses a single occupied-name set that can accidentally remove the local conflicting name. This plan wires the Core sharing contract into the GUI and makes rename checks source-aware.
Output: App save context construction, source-aware rename validation, and focused tests for EXT-03 and EXT-05 blockers from `04-VERIFICATION.md`.
</objective>
