<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Profile JSON file path → Profiles workspace | User-selected local files may be missing, locked, unreadable, or maliciously named. |
| Profile source row selection → manager commands | UI row selection is untrusted transient state when an unsaved editor buffer is active. |
| Catalog refresh/search → active editor | Runtime catalog changes and search filtering must not discard unsaved local user data silently. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-12-01 | Tampering | `SaveSelectedProfileAsync` | mitigate | Save only `Editor.BuildProfile(ProfileSourceKind.LocalCustom, filePath)` after validation; bundled rows remain read-only and existing local file path is reused only for selected LocalCustom rows. |
| T-04-12-02 | Repudiation | `TrySelectProfileAsync` | mitigate | Track committed selection and restore it when discard is declined so row-scoped commands reflect the visible editor. |
| T-04-12-03 | Denial of Service | `ImportProfilesAsync` / `ExportSelectedProfileAsync` | mitigate | Catch expected `IOException`/`UnauthorizedAccessException` and report actionable `StatusMessage` text without crashing the command pipeline. |
| T-04-12-04 | Information Disclosure | `ExportSelectedProfileAsync` | accept | Export remains limited to explicit LocalCustom/EmbeddedProject selections from prior plans; this plan only changes failure handling and does not broaden export scope. |
</threat_model>
