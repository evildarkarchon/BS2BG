<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User search text → profile editor visible row projections | Search text controls what is displayed but must not mutate source profile rows or saved output. |
| AXAML compiled bindings → ViewModel collections | UI bindings must target existing strongly typed ViewModel properties to avoid runtime binding failures. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-13-01 | Tampering | `RefreshVisibleRows` | mitigate | Populate visible projections only; keep `BuildDefaults`, `BuildMultipliers`, and `BuildInverted` reading source collections so filters cannot remove profile data. |
| T-04-13-02 | Denial of Service | Profile editor filtering | accept | Filtering is in-memory over existing ObservableCollections; no file, network, or unbounded background work is added. |
| T-04-13-03 | Information Disclosure | Filtered profile rows | accept | Search only narrows local visible rows already loaded into the editor; it does not expose external data. |
</threat_model>
