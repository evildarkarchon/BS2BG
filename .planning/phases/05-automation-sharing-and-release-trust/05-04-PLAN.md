Formal algorithm spec required by review: strategy scope is an explicit eligible NPC list supplied by callers. CLI/core default is all `project.MorphedNpcs`; GUI Plan 05 will deliberately pass all rows for cross-machine determinism and show the count before apply. `ComputeEligibility(ProjectModel, AssignmentStrategyDefinition, IReadOnlyList&lt;Npc&gt; eligibleRows)` is the single non-mutating source used by both `Apply` and diagnostics; duplicate eligibility logic is forbidden. Plain export validation must not block only because a saved strategy has no-eligible rows unless the caller is applying that strategy in the current operation; D-15 blocks assignment for that NPC, not unrelated export of already-assigned morphs.

Deterministic PRNG requirement from reviews: persisted seed replay MUST NOT use `System.Random(seed)`. Implement a small pinned algorithm in `DeterministicAssignmentRandomProvider` behind `IRandomAssignmentProvider` (recommended: Mulberry32 with uint state, documented in XML comments) and add reference-vector tests for seed 0, 1, and 123. Acceptance criteria must grep production files to ensure no `new Random(seed)` appears in strategy replay code.

Weighted semantics: for each NPC, select matching rules in stable order by rule Name OrdinalIgnoreCase, then original rule order as tie-breaker. A matching rule is one whose RaceFilters is empty or contains `Npc.Race` OrdinalIgnoreCase and whose PresetNames resolve to existing project presets. Draw one integer from the provider in `[0,totalWeightUnits)` where totalWeightUnits is the sum of positive integer weight units after converting Weight to fixed two-decimal units (`Math.Round(weight * 100, MidpointRounding.AwayFromZero)`). Document that `Weight = 0.005` becomes one unit because Phase 5 rounds half-up to two decimal units; values that round to 0 units do not participate. Pick the first cumulative rule exceeding the draw, then pick a preset from that rule's eligible PresetNames using the same provider seam. If the only rules matching an NPC after race narrowing all quantize to zero units, that NPC is blocked per D-15; do not fall back to all presets or to non-matching rules. All-zero/invalid global strategy weights were rejected by Plan 03.

Round-robin semantics: always start at index 0 for each full Apply call, then advance one preset per eligible NPC in stable eligible NPC order; do not resume from previous run state. GroupsBuckets semantics: bucket membership is rule-predicate based, not game lookup. A rule matches an NPC when `BucketName` is non-empty and RaceFilters is empty or contains `Npc.Race` OrdinalIgnoreCase; matching rules are ordered by BucketName then rule Name OrdinalIgnoreCase. Eligibility is the union of matching rules' PresetNames in project preset order. If no bucket rule matches or all preset names are unknown, that NPC is blocked per D-15. Addresses review concern: empty RaceFilters for GroupsBuckets means match any imported race, while non-empty RaceFilters narrow by imported race text.
