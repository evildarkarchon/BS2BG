## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User filesystem→bundle zip target | Existing user-created bundle may be replaced by a new archive. |
| Shared project/custom profile data→generated bundle outputs | Embedded/local custom profile JSON influences generated BodyGen/BoS bytes. |
| CLI user input→process exit contract | Missing/malformed project paths and filesystem failures cross into automation scripts. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-10-01 | Tampering | PortableProjectBundleService.Create | mitigate | Write replacement zip to same-directory temp path and only commit with File.Replace/File.Move after archive close; never delete prior final path on failure. |
| T-05-10-02 | Information Disclosure | CLI bundle failure messages | mitigate | Map expected exceptions to concise stderr without stack traces, exception type names, full profile asset paths, or private roots. |
| T-05-10-03 | Tampering | Bundle generation catalog | mitigate | Build request-scoped catalog from bundled profiles plus referenced custom profiles from ProjectModel.CustomProfiles/ProjectSaveContext before validation/generation. |
| T-05-10-04 | Denial of Service | Temp bundle artifacts | accept | Abrupt process termination can leave temp files; normal managed failures use best-effort temp cleanup and preserve final bundle. |
</threat_model>
