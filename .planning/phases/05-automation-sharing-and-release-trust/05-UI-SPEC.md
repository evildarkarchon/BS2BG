| Property | Value |
|----------|-------|

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Tight row gaps, command-palette/list item margins, badge-to-label gaps |
| sm | 8px | Compact button gaps, filter popover padding, row spacing, inline control separation |
| md | 16px | Default panel padding, section margins, empty-state padding, card padding |
| lg | 24px | Empty-state outer margin and large contextual separation |
| xl | 32px | Major workspace grouping when a new phase surface needs more separation than existing tabs |
| 2xl | 48px | Reserved for page-level empty states only |
| 3xl | 64px | Reserved; do not introduce in dense data-management panes without checker approval |

Exceptions: existing resizable splitters remain 4px; existing compact table/list row margins may use `0,4`; existing fixed fields (`Width="64"`, `Width="108"`, `Width="140"`, popover `Width="240"`) may remain where already established. Minimum pointer target for new iconless controls is 32px high; do not add icon-only controls in this phase.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `BS2BGWindowBackgroundBrush` — Light `#F7F9FC`, Dark `#11151A` | Window background and broad workspace surface |
| Secondary (30%) | `SystemControlBackgroundChromeMediumBrush` | Cards, sidebars, popovers, detail panels, strategy editor panels, bundle preview panels |
| Accent (10%) | `BS2BGFocusBrush` / active-filter resources — Light `#005FB8`, Dark `#7CC4FF` | Focus ring, active filter badges, selected strategy badges, checksum verified badge, deterministic seed active state |
| Destructive | `SystemControlErrorTextForegroundBrush` | Destructive action labels/errors only: overwrite refusal/error, delete strategy/group/rule, clear strategy, failed verification |

Accent reserved for: focus indicators, active filter badges, selected strategy/rule badges, verified checksum/signing status, and primary CTA emphasis. Do not use accent for all interactive controls or as the only indicator of trust state.

Semantic support colors already available:
- Info: `BS2BGInfoBackgroundBrush`, `BS2BGInfoBorderBrush`, `BS2BGInfoForegroundBrush` for neutral fallback, bundle path-scrubbed notices, unsigned-but-checksummed release notes, and empty states.
- Warning: `BS2BGWarningBackgroundBrush`, `BS2BGWarningBorderBrush`, `BS2BGWarningForegroundBrush` for overwrite risk, unsigned release verification guidance, and non-blocking caution diagnostics.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Create Portable Bundle |
| Empty state heading | No assignment strategy configured |
| Empty state body | Choose seeded random, round-robin, weights, race filters, or groups to make assignments reproducible in the GUI, CLI, and shared bundles. |
| Error state | Strategy rules leave NPCs without eligible presets. Adjust race filters, weights, or groups before applying assignments. |
| Destructive confirmation | Delete strategy rule: Delete this rule? Existing assignments stay unchanged until you apply a strategy again. |

Additional required copy:
- CLI surface label: `Automation CLI`.
- Bundle preview label: `Portable bundle preview`.
- Bundle privacy note: `Bundle reports use relative paths and source filenames only.`
- Overwrite refusal: `Target files already exist. Enable overwrite to replace them.`
- Unsigned release trust note: `This build is unsigned. Verify the SHA-256 checksum before running it.`
- Signing success note: `Signature verified. Checksum matches the packaged sidecar.`
- No eligible assignment row: `No eligible preset after strategy rules`.
- Packaged docs boundary: `BS2BG generates files and setup guidance; it does not edit game plugins.`

Tone: factual, local/offline, trust-focused, and non-alarming. Use `Blocker`, `Caution`, and `Info` labels consistently with Diagnostics. Avoid `experimental` for Fallout 4 profile status in app UI.

## Phase 5 Interaction Contract

### Navigation and placement
- Keep the existing top-level workspace model. Add Phase 5 GUI surfaces inside the existing shell; do not replace the `Templates`, `Morphs`, `Diagnostics`, or `Profiles` workflows.
- Assignment strategy configuration belongs in the `Morphs` workspace near NPC bulk assignment controls because it changes NPC assignment behavior.
- Portable bundle creation belongs in the shell export/share workflow and may be exposed from the File menu or a clearly labeled bundle action; it must show a preview before writing the zip.
- Setup/troubleshooting guidance remains packaged docs only for Phase 5. Do not add an in-app wizard or new Help-menu UI solely for BodyGen/BodySlide/BoS guidance.

### Assignment strategy editor
- Strategy type selector labels: `Seeded random`, `Round-robin`, `Weighted`, `Race filters`, `Groups / buckets`.
- Always show a read-only summary line: `Saved in project; reproducible in CLI and bundles.`
- Seed field label: `Seed`; helper text: `Same project, strategy, eligible rows, and seed produce the same assignments.`
- Race filter helper text: `Matches imported NPC Race text only; no game-data lookup.`
- Rule gaps must render as explicit row text (`No eligible preset after strategy rules`) plus a diagnostics finding; never silently fall back to all presets.
- Applying a strategy is undoable and dirty-tracked. CTA label: `Apply Strategy`.
- Rule deletion requires confirmation only when the rule has non-empty criteria or affects existing saved strategy configuration.

### Portable bundle preview
- Preview must list the deterministic zip layout before write: `project/`, `outputs/bodygen/`, `outputs/bos/`, `profiles/`, `reports/`, `manifest.json`, checksum entries.
- Preview must explicitly state whether generated outputs are included and which output intents are selected.
- Profile copy summary must say `Referenced custom profiles only` when applicable.
- Privacy status must be text-visible, not color-only: `No absolute paths in manifest/report` or `Path privacy check failed`.
- The write action must refuse existing zip targets unless the user chooses overwrite through an explicit checkbox or confirmation.

### CLI and release trust presentation
- In-app CLI references should be copyable text snippets, not an interactive terminal.
- CLI command examples use monospace 13px and avoid wrapping paths mid-token when possible.
- Release verification states must be explicit text badges: `Signed`, `Unsigned`, `Checksum verified`, `Checksum mismatch`, `Signature unavailable`.
- Unsigned is a caution state, not a blocker, when SHA-256 sidecars and unsigned verification docs are present.

### Accessibility and compiled binding contract
- Every new AXAML root and every `DataTemplate` must declare `x:DataType`.
- Every actionable control must set `AutomationProperties.Name`; include accelerator keys when a menu item has a shortcut.
- Trust states must use visible text labels in addition to color.
- New resizable two-pane layouts must include `GridSplitter` with 4px splitter columns/rows and nonzero min sizes for adjacent regions.
- Do not introduce ReactiveWindow, RelayCommand, dispatcher calls in ViewModels, or WPF-only XAML concepts.
