### Critical Integration Gaps

| Gap | Affected Requirements | Evidence | Impact |
|---|---|---|---|
| CLI generation does not activate embedded/custom project profiles | AUTO-01, EXT-03, EXT-04, PROF-03, PROF-04 | `src/BS2BG.Cli/Program.cs:179-189` creates `HeadlessGenerationService` with `new TemplateProfileCatalogFactory().Create()`. `src/BS2BG.Core/Automation/HeadlessGenerationService.cs:61-99` validates and generates with that catalog. `ProjectFileService.Load` loads `ProjectModel.CustomProfiles`, but headless generation never composes them into a request catalog. | A `.jbs2bg` project that references an embedded custom profile can generate correctly through GUI/bundle paths but CLI `generate` falls back to bundled profile semantics. |
| Saved deterministic assignment strategies are not replayed by CLI or bundle generation | AUTO-03, AUTO-01, AUTO-02, WORK-02, WORK-03, WORK-04 | `HeadlessGenerationService.cs:88-90` and `PortableProjectBundleService.cs:263-265` call `MorphGenerationService.GenerateMorphs(project)` directly. `MorphGenerationService` serializes current assignments and does not inspect `ProjectModel.AssignmentStrategy`. | Automation reproduces already-mutated assignments only; saved strategy configuration is not applied before CLI/bundle output. |
